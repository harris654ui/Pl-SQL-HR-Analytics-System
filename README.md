# PL-SQL-HR-Analytics-System

## Project Overview

The **PL-SQL-HR-Analytics-System** is a PL/SQL-based analytics project built on top of Oracle's built-in **HR Schema**. The system performs employee analytics and turnover risk prediction using data extracted from:

* `hr.employees`
* `hr.job_history`

This project demonstrates:

* Advanced PL/SQL package development
* Analytical functions
* Risk scoring logic
* Aggregated department reporting
* Basic ETL (Extract, Transform, Load) process

### ETL Component

A custom table was created for this project:

`hr.turnover_predictions`

Data is:

* **Extracted** from `hr.employees` and `hr.job_history`
* **Transformed** using business logic (tenure, promotion count, salary comparison)
* **Loaded** into the custom prediction table

```sql
CREATE TABLE hr.turnover_predictions (
    prediction_id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    employee_id   NUMBER REFERENCES hr.employees(employee_id),
    risk_score    NUMBER,
    risk_level    VARCHAR2(20),
    calculated_on DATE DEFAULT SYSDATE
);
```

This simulates a simplified HR analytics and turnover prediction system commonly used in enterprise environments.

## Package Overview: `hr_analytics`

All logic is encapsulated inside a PL/SQL package named:

`hr_analytics`

The package contains analytical functions and reporting procedures.

---

### Functions

#### 1. `months_with_comp(p_employee_id)`

Calculates the total number of months an employee has worked across all job history records.

* Uses `months_between(end_date, start_date)`
* Aggregates total tenure in months
* Data source: `hr.job_history`

Purpose:
Used to measure employee experience and tenure for risk calculation.

---

#### 2. `job_history(p_employee_id)`

Counts how many job history records an employee has.

Purpose:
Used as an indicator of promotions or role changes.

---

#### 3. `salary_growth(p_employee_id)`

Compares an employee’s current salary with the average salary of their department.

Logic:

* Retrieves employee salary
* Retrieves department average salary
* Returns:

  * `1` if above department average
  * `0` if below average

Purpose:
Determines salary competitiveness within the department.

---

#### 4. `turnover_risk(p_employee_id)`

Calculates a numeric turnover risk score based on:

* Total tenure (months worked)
* Promotion count
* Salary comparison
* Long tenure without sufficient growth

Risk score is built using weighted rules.

Purpose:
Core analytics engine that determines employee turnover likelihood.

---

### Procedures

#### 1. `calculate_all_turnover_risk`

Loops through all employees and:

* Calls `turnover_risk`
* Determines risk level:

  * High (>= 70)
  * Medium (>= 40)
  * Low (< 40)
* Inserts results into `hr.turnover_predictions`
* Commits the transaction

Purpose:
Automates turnover scoring for the entire organization.

This procedure performs the **Load** part of the ETL process.

---

#### 2. `department_analytics_report`

Generates a department-level HR analytics summary.

For each department, it calculates:

* Total employee headcount
* Average salary
* Highest salary
* Lowest salary
* Average tenure (in months)
* Number of high-risk employees

Outputs results using `DBMS_OUTPUT`.

Purpose:
Provides strategic HR insights at department level.

## Installation / How to Use

### Step 1: Requirements

* Oracle Database installed
* Access to the HR sample schema
* TOAD for Oracle or any PL/SQL IDE
* Privileges to create tables, packages, and procedures

---

### Step 2: Create Custom Table

Run the `turnover_predictions` table creation script.

---

### Step 3: Create Package

Execute:

1. Package Specification
2. Package Body

Ensure no compilation errors.

---

### Step 4: Run Turnover Risk Calculation

```sql
BEGIN
   hr_analytics.calculate_all_turnover_risk;
END;
/
```

This will populate the `hr.turnover_predictions` table.

---

### Step 5: Run Department Analytics Report

Enable server output:

```sql
SET SERVEROUTPUT ON;
```

Then execute:

```sql
BEGIN
   hr_analytics.department_analytics_report;
END;
/
```

This will print the analytics report in the console.
